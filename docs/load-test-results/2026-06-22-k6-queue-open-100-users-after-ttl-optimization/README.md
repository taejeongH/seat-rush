# 대기열 TTL 및 polling 최적화 후 100명 재측정

## 조건

- 대상: 배포 EC2 `t4g.large`, HTTPS API Gateway 경유
- 시나리오: 연습 세션 생성 후 대기열 진입, 순번 조회, entryToken 발급
- 사용자 수: 100명
- 워밍업: 10명, 오픈 대기 5초, 1회 실행
- 본 측정: 오픈 대기 5초, 3회 실행
- polling: 2초, heartbeat: 10초

모든 실행에서 100명 전원이 entryToken을 발급받았고 HTTP 실패는 없었습니다.
